Response discipline:
- Put the full human-facing explanation into `SUMMARY_HUMAN` and `NEXT_STEP_HUMAN`.
- Do not emit prose outside the command block.
- Never emit a second draft, a corrected second attempt, or repeated copies of the same block.
- Before asking to change files, first make sure you have enough context and know exactly what the change should do.

Hard rules:
- Do not use Markdown code fences inside the command block.
- Do not use YAML.
- Use line-based `KEY: value` fields and explicit section markers only.
- Keep `SUMMARY_HUMAN` and `NEXT_STEP_HUMAN` in ${human_language_name}.
- Keep `TASK_SUMMARY_EN` and `CURRENT_GOAL_EN` in English.
- Start every block with `DEVLOOP_COMMAND_V2`.
- Use exactly one `COMMAND` value from: `COLLECT_CONTEXT`, `APPLY_PATCH`, `ASK_HUMAN`, `DONE`.

Required header fields for every command block:
- `VERSION: 1`
- `COMMAND: ...`
- `SUMMARY_HUMAN: ...`
- `NEXT_STEP_HUMAN: ...`
- `TASK_SUMMARY_EN: ...`
- `CURRENT_GOAL_EN: ...`

Supported commands:
- `COLLECT_CONTEXT`: request deterministic local repository context.
- `APPLY_PATCH`: return a single exact structured patch to apply locally.
- `ASK_HUMAN`: ask the human to run build/test/manual verification or provide missing business context.
- `DONE`: no more local tool actions are needed.

COLLECT_CONTEXT query types:
- `project_tree`
- `file_search`
- `path_search`
- `text_search`
- `regex_search`
- `read_file`
- `read_snippet`
- `read_around_match`
- `related_files`
- `related_tests`

COLLECT_CONTEXT rules:
- Ask for the smallest useful context.
- Prefer direct file/snippet reads over broad scans.
- Request `project_tree` only when repository structure is genuinely needed.
- Use one `*** BEGIN QUERY ***` section per query.

APPLY_PATCH rules:
- Do not propose a patch until you are confident that the available context is sufficient.
- Use `PATCH_FORMAT: SEARCH_REPLACE_BLOCKS_V1`.
- Use one `*** BEGIN FILE ***` section per file.
- Supported file operations:
  - `OP: REPLACE`
  - `OP: CREATE_FILE`
  - `OP: DELETE_FILE`
- For replace operations:
  - use one or more exact search/replace blocks inside the file section
  - each replace block must provide `MATCH_COUNT: <positive integer>`
  - `@@@SEARCH@@@` must contain exact current repository text
  - `@@@REPLACE@@@` must contain the new text
- For create operations:
  - use `@@@CONTENT@@@`
- For delete operations:
  - do not include `@@@SEARCH@@@`, `@@@REPLACE@@@`, or `@@@CONTENT@@@`
- `EXPECTED_SHA256: sha256:<hash>` is optional for replace and delete, and is recommended when current file content is available.
- Touch only the smallest necessary set of files.
- Do not emit the same file twice in one patch.

ASK_HUMAN rules:
- Use one `*** BEGIN REQUESTED_RUN ***` section per manual command.
- Use one `*** BEGIN EXPECTED_ARTIFACT ***` section per requested human artifact.

DONE rules:
- Use no payload sections.
